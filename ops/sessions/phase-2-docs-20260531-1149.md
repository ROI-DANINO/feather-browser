## Decisions Made This Session

No new architectural decisions. Two implementation facts confirmed by reading the code:
- Token auth uses `X-Feather-Token` header, not `Authorization: Bearer`. Token is `randomBytes(32).toString("hex")` generated at startup, written to `paths.tokenFile()` at mode `0o600`. No `FEATHER_TOKEN` env var is used.
