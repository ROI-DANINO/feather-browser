### IManager / getPage() type mismatch (routes.ts)
`FeatherSession.getPage(pageId)` returns `Page` (Playwright) directly. Four command handlers (Navigate, Snapshot, Extract, Screenshot) each define a local `IManager` interface expecting `getPage()` to return `{ pageId: string; page: Page }`. The mismatch is papered over with `manager as any` at the constructor call sites in routes.ts. This works at runtime but suppresses type safety. Consider aligning the interfaces or changing handler constructors to accept `SessionManager` directly in Phase 3.
