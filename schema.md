# Schema

## Operating Files

- `ROADMAP.md`: destination, phase milestones, and exit criteria.
- `PROGRESS.md`: current state, decisions, open questions, and next action.
- `ops/phase.md`: machine-readable current phase state.
- `ops/tasks.md`: source of truth for current phase tasks.
- `context/active.md`: resume context for `/start`.
- `ops/sessions/`: `/stop` handoff files.
- `raw/_inbox/`: rough notes, dropped links, and session checkpoints.
- `.remember/remember.md`: short handoff for the next session.

## Rule

Only the current phase gets detailed tasks. Future phases stay in `ROADMAP.md` until the prior phase is complete.
