# Session 5.0.0 — Local Control-Plane And Capability Gate

**Phase:** 5.0 — Agent Interface And Safety Gate
**Status:** Future — **the first Phase 5 work.** Gates everything downstream (warmed CDP attach, MFA
local routes, `vaultRef` activation).
**Classification:** Security foundation. Implements ADR-0010.

> This session exists because the 2026-06-07 council review (5/5) found Feather was about to expose
> its highest-privilege surfaces before any machinery governed them. It builds the capability/security
> model **first**, so interop happens *through* that model. See
> `docs/specs/adr-0010-local-control-plane-capability-model.md` and
> `research/2026-06-07-council-design-review.md`.

## Goal

Build the local control-plane / capability model from ADR-0010: privilege tiers, the capability-grant
primitive for dangerous operations, transport hardening (`Origin`/`Host` hook), and the session-hold
primitive — so a local caller can prove it may touch a warmed session, and dangerous operations are
short-lived, scoped, human-approved, and audited.

## Tasks

- **Transport hardening first (cheap, highest-leverage):** add a global `Origin` / `Referer` /
  `Host` validation hook (`src/transport/`) that rejects cross-origin browser-initiated requests and
  mismatched `Host`. Keep loopback bind. This closes the worst concrete hole (the MFA CSRF drive-by)
  and can land as a standalone hardening step.
- **Session-hold primitive:** add refcounted holds with a `reason`
  (`"mfa" | "human-approval" | "cdp-attach" | "shutdown"`) to the session, plus a policy layer that
  *observes* holds. Nothing toggles stealth directly. (This is what breaks the MFA → Stealth
  dependency and lets Stealth move last.)
- **Capability-grant primitive:** mint single-use / short-TTL grants scoped to one `sessionId` + one
  capability (`cdp-attach` | `vault-unlock` | `cookie-export`); require human approval at grant time;
  auto-revoke on session close / MFA open / shutdown.
- **Audit events:** `grant.requested` / `granted` / `denied` / `used` / `expired` / `revoked`
  (redacted) on the logging bus; decide durable audit surface (Open Q4 in ADR-0010).
- **Dangerous-mode policy:** off by default; explicit opt-in + per-grant approval.
- **Human-approval surface (no shell yet):** decide console + local approval page (reuse the MFA
  local-page pattern) vs. Telegram (ADR-0010 Open Q2).
- Resolve ADR-0010's open questions with Roi; move ADR-0010 to **Accepted**.

## Expected output

- A working capability gate: token tiers, capability grants, `Origin`/`Host` hook, hold primitive,
  audit events. ADR-0010 accepted.

## Definition of done

- Dangerous operations cannot be performed with the bearer token alone.
- The `Origin`/`Host` hook rejects a simulated CSRF drive-by in tests.
- Holds are refcounted and observed by the policy layer; no direct stealth toggling exists.
- Full unit/integration/typecheck gates pass; API docs updated.

## Risks / blockers

- Scope: this is foundational; keep it a *gate*, not an agent framework.
- Don't pre-commit cross-module type contracts (council) — specify behaviors/seams; fix concrete
  types as each consuming module is built.

## Do not do yet

- Do not implement warmed CDP attach, MFA hardening, or vault release here — this session builds the
  gate they pass through.
