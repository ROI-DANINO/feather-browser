# Feather Roadmap and Future Plans

Feather is a local Chromium browser runtime that lets AI agents operate real browser sessions through a controlled HTTP API. This source explains the v1, v2, and v3 roadmap and how current proof points lead into future safety work.

## Roadmap Summary
Feather targets progressive capability milestones over three phases.

## v1: It Runs Errands for Me
- Built: Core input commands, snapshots, observability.
- Verified: End-to-end errand running (Instagram test, showcase suite).

## v2: It Survives Scary Sites, Safely
- Planned: Gate A (Capability gate), Gate B, Identity Model, MFA Handler, Stealth stack.
- Planned: capability/safety gate -> Identity -> MFA -> warmed-profile attach -> Stealth hardening.

## v3: The Polished Product
- Planned: Visual Zen-style browser shell.
- Planned: Agent Runtime / ecosystem interop.

## Current Next Choices
- Planned: Observe-loop measurement.
- Planned: Start v2 Gate A.

## Deferred and Parked Work
- Deferred: Cross-origin iframe descent (spec §16).
- Parked: pi_agency integration.
