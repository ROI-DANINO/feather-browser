# Feather v3 — "The polished product"

> Front door: [`feather.md`](../../feather.md). Sibling roadmaps: [v1](v1.md), [v2](v2.md).
> Execution index (phases/sessions/ADRs): [`ROADMAP.md`](../../ROADMAP.md).

**Goal:** the nice daily-driver browser window + opening Feather to the wider ecosystem.
**Lowest priority** — until v3 you use plain Chromium, Feather-style, behind the scenes.

## Features

- [ ] **Visual browser window** — a minimalist, Zen-inspired shell that owns the long-running human
      browsing context (the human side of the Cookie Mine). Phase 4b.
      Read: [`adr-0007`](../specs/adr-0007-phase-4-shell-sequencing.md),
      [`adr-0009`](../specs/adr-0009-shell-stack.md) (shell stack — CANDIDATE).
- [ ] **Open Feather to other agent frameworks** (ecosystem interop) — Session 5e. **This absorbs the
      old 4a.7 "let outside tools attach over CDP" idea** — correctly placed *last*, because letting
      others drive Feather is the opposite of native. Read:
      [`adr-0005`](../specs/adr-0005-agentic-north-star.md),
      [`adr-0006`](../specs/adr-0006-agent-interface-neutrality.md).
- [ ] **True perception / generalized workflows** (north star) — reason about and operate completely
      unfamiliar sites. May start informing v2; lands fully here.

## Constraint

[`adr-0009`](../specs/adr-0009-shell-stack.md) (shell stack: Tauri vs GTK4) stays **CANDIDATE** until
the Casilda + headed-Chromium latency/input spike runs on the Fedora/Wayland box. Don't commit the
shell stack before that spike.

## The caught mistake, parked correctly here

The original Session 4a.7 planned to let outside tools drive Feather over CDP as a *side runtime* —
the opposite of the native-build philosophy. It is not deleted; it is **moved to v3 ecosystem interop**
(Session 5e), where exposing Feather to others is the actual goal.
