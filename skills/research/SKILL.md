# Research

## Overview

Mid-work research pass. Investigates a specific topic and surfaces only what's relevant to the current task — then stops for confirmation before acting on findings.

## Steps

1. **Clarify** — if the topic is vague, ask one focused question before searching.
2. **Search** — look for docs, changelogs, issues, articles, or specs. Prefer primary sources (official docs, GitHub issues, spec pages) over summaries.
3. **Synthesize** — separate signal from noise in 3–5 bullet points.
4. **Relate** — does this change anything in progress? Flag if it affects the current task, a planned phase, or an assumption in the codebase.
5. **Confirm** — ask: "Want me to apply any of this — update a task, adjust the approach, or note it in PROGRESS.md?" Do not touch files without confirmation.
