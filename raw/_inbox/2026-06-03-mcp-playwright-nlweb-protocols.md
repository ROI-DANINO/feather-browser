# MCP, Playwright MCP, and NLWeb

## Purpose

Capture ecosystem context around protocol-level developments relevant to future Feather planning.

---

## MCP

Observed themes:

- growing ecosystem adoption
- tool interoperability
- model-to-tool communication

Potential relevance:

Future Feather integrations may eventually interact with MCP ecosystems.

---

## Playwright MCP

Observed themes:

- browser control through MCP
- agent/browser interoperability

Potential relevance:

A useful reference point when evaluating future integration approaches.

Not necessarily a reason to commit early.

---

## NLWeb

Observed themes:

- agent-friendly web interfaces
- natural-language access patterns
- structured interaction models

Potential relevance:

Suggests a possible future where browser automation is not the only path for agent interaction.

---

## Strategic observation

There appears to be tension between:

1. Agents operating websites through browsers.
2. Agents operating websites through APIs/protocols.

Future systems may use both.

---

## Planning caution

Protocol ecosystems appear to be evolving quickly.

Potential implication:

Remain informed without locking architecture too early.
