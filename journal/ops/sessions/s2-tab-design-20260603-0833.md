## Decisions

1. **TAB_UPDATED = settled updates only.** One event per navigation, fired after
   `domcontentloaded` so the title is real. No loading-spinner pulse (deferred to Phase-4 shell if
   ever wanted). Mechanism: main-frame `framenavigated` + `waitForLoadState("domcontentloaded")` +
   supersede guard; covers SPA `pushState`. Payload `{ pageId, url, title, loadState }`.
   - Key finding driving this: `page.title()` is unreliable at `framenavigated` time (title not yet
     parsed), so a settled read is required regardless.
2. **Dup-registration fix = idempotent `addPage` keyed on the `Page` object.** Reverse map
   `Page → pageId`; `openTab()` stops assigning its own id. Eliminates the two-IDs bug and the
   listener-vs-`openTab` race.
3. **Keep `TAB_OPENED` and `TAB_CREATED` distinct** (intent/audit vs lifecycle). Not collapsing —
   would be an event-contract change / scope creep.
4. **Defer `FEATHER_CHROMIUM_PATH`** out of this plan — different theme (weight), gated on a
   `sudo dnf install chromium` spike. Follow-on.
5. **S2 implementation plan scoped to 3 items** (dup-reg fix → TAB_UPDATED → observability).
