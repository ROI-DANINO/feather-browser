# Open Source Browser Agents

## Purpose

Survey nearby open-source projects and communities that appear relevant to Feather's long-term direction.

---

## OpenClaw / Moltbot

Observed themes:

- local-first execution
- self-hosted agents
- tool integrations
- user-owned infrastructure

Potential relevance:

Demonstrates demand for local agent runtimes rather than cloud-only agents.

Questions:

- What parts of OpenClaw's architecture are runtime concerns versus agent concerns?
- Which responsibilities should Feather explicitly avoid taking on?

---

## Browserbase + Stagehand

Observed themes:

- browser automation infrastructure
- Playwright-centered workflows
- AI-friendly browser operations

Potential relevance:

Useful reference point for browser-control APIs and observability patterns.

---

## browser-use

Observed themes:

- task completion through browsers
- agent/browser integration

Potential relevance:

Shows how developers currently consume browser automation as an agent capability.

---

## LiteWebAgent

Observed themes:

- web agents
- planning
- memory
- browser interaction strategies

Potential relevance:

Interesting for future agent-facing design research.

---

## Agent S / computer-use style systems

Observed themes:

- GUI automation
- hierarchical planning
- computer-use workflows

Potential relevance:

Highlights the distinction between:

- browser-level automation
- operating-system-level automation

Feather currently appears closer to browser-level infrastructure.

---

## Hugging Face Open Computer Agent

Observed themes:

- open-source computer use
- transparency
- experimentation

Potential relevance:

Useful as a signal of where open communities are exploring agent control.

---

## Synthesis

A recurring pattern:

Many projects focus on the agent.

Feather may be stronger focusing on the runtime and browser substrate first.
