# Archived Next Bundles

`/stop` moves consumed `journal/context/next.md` content here instead of deleting it.

Use:

- `journal/archive/next/YYYY-MM-DD/HHMM-<short-session-name>.md` for a single pending `/next` entry
- `journal/archive/next/YYYY-MM-DD/HHMM-stop-bundle-<short-session-name>.md` for a multi-entry bundle

After archiving, `/stop` resets `journal/context/next.md` to an empty active buffer.
